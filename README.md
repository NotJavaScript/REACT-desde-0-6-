# Proyecto 6: Optimización de Componentes y Estrategias de Despliegue

Este repositorio marca la evolución del proyecto entre el quinto y el sexto bloque de formación de **React JS desde 0**. En esta fase, aunque la interfaz se mantiene constante, el trabajo se ha centrado en la "capa invisible" del desarrollo la eficiencia del renderizado y el ciclo de vida de puesta en producción.

El objetivo de esta etapa es asegurar que la aplicación no solo funcione, sino que sea altamente eficiente y esté disponible para el mundo exterior.

---

## 🚀 Funcionalidades y Aprendizaje

### 1. Optimización del Rendimiento

A diferencia de etapas anteriores, los cambios aquí son estructurales y de comportamiento interno del framework.

* **Uso de React.memo:** Implementación de componentes de orden superior (HOC) para evitar re-renderizados innecesarios, memorizando el resultado de la renderización en función de las props.
* **Perfilado de Renderizado:** Análisis de qué componentes necesitan realmente actualizarse, mejorando la fluidez general de la aplicación bajo carga de datos.

### 2. DevOps y Deployment Profesional

Se ha trascendido el entorno de desarrollo local para entender cómo escalan las aplicaciones reales en la industria.

* **Despliegue con Vercel:** Configuración y puesta en marcha del proyecto en la plataforma de hosting líder en el ecosistema de React/Next.js.
* **Workflow de Producción:** Aprendizaje sobre el proceso de *build* y despliegue continuo (CD) que utilizan empresas de primer nivel como Airbnb o GitHub.

---

## 📝 Notas sobre el entorno

Para esta etapa, el foco ha estado en herramientas que garantizan la escalabilidad:

* **Memorización de UI:** Aplicación de técnicas para controlar la profundidad de las actualizaciones en el DOM virtual.
* **Infraestructura en la Nube:** Configuración de variables y entornos de producción para que la app sea accesible vía URL pública.
* **Estándares de la Industria:** Adopción de flujos de trabajo profesionales para la distribución de aplicaciones web modernas.
